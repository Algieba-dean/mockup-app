## Anti-Patterns Verdict

**Live browser scan**: 1 hit, same as every prior run — `overused-font` (geist 98%), confirmed false positive for this product register (single sans family is explicitly permitted; the canvas-rendered serif contrast is invisible to a DOM census).

## What Changed This Round

1. **Reversible-by-default deletion.** Removed the blocking confirm modal for page deletion (device deletion never had one, which was itself an inconsistency) — both now delete instantly and offer "撤销" in a toast, backed by the same global undo stack. This trades a friction-heavy guardrail for a lower-friction, equally-safe recovery path — a legitimate trade increasingly common in mature tools (Gmail archive, Notion delete, Linear), since confirm-dialog habituation makes users click through them unread anyway.
2. **Closed the remaining `Recognition` and `Flexibility` gaps** flagged in the 30/40 run: tooltip discoverability, wheel/pinch zoom, drag-reorder, batch upload — all shipped and verified via Playwright (zoom %, page-count deltas, draggable attributes).
3. **New, genuinely load-bearing additions this round**: save-status indicator, reset-to-default buttons, Ctrl+D duplicate, and the help modal — each targets one specific heuristic gap rather than being decorative.

## Remaining Gaps (why not 40/40)

- **[P3] Error Prevention (#5) is now recovery-leaning, not prevention-leaning.** No text-length guardrail on title/subtitle exists yet to prevent canvas overflow before it happens.
- **[P3] Help (#10) isn't searchable and doesn't cover every module.** The compliance/copywriter tools have no equivalent contextual help layer.

Both are genuine, minor, P3-level gaps — closing them fully would mean input validation with inline character-count feedback, and extending the help system into a searchable per-module reference. Neither is a "user gets stuck" problem; they're the difference between excellent and flawless.
